# 产品数据消失问题 —— 根因诊断 + 修复方案

## 一、到底是什么原因导致 256 个产品变成 5 个、产品详情页打不开

我把你上传的 `admin.js`、`index.html`（后台首页）、`product_edit.html`、`products-data.js` 全部读了一遍，代码里其实已经有前人（可能是之前某次修复）留下的中文注释，直接指出了病灶，我把链路串起来给你看：

### 1. 系统里同时存在 **三份"产品数据"**，谁也不完全信任谁
- `aec-project/products-data.js` —— 前端网站真正读取的那份，也是你每次保存"应该"落地的地方。
- `admin/products-data.js` —— 后台自己目录下**打包携带的一份旧副本**（很可能是很久以前生成的，或者是模板初始值）。`admin/index.html` 用 `<script src="products-data.js">` 加载的，**默认加载的就是这一份旧的**，不是 aec-project 里的。
- 浏览器 `localStorage`（`aecAdmin_products` 等 key）—— 你每次在后台修改后，先写进这里做"草稿缓存"。

### 2. 后台每次打开时的读取顺序，正常情况下没问题：
连接文件夹 → 从磁盘 `aec-project/products-data.js` 读最新数据 → 覆盖 localStorage → 显示。

### 3. 但只要下面任意一种情况发生，后台就会"看到"一份很小的数据：
- 忘记重新"选择网站文件夹"（操作手册里也写了，每次都要重选，这是浏览器安全限制，很容易忘）；
- 换了浏览器 / 换了电脑 / 清过缓存，导致 `localStorage` 是空的；
- 选择文件夹后，磁盘文件读取解析失败（哪怕只是格式稍有出入）。

这些情况下，后台会退回去用 `admin/products-data.js`（旧副本）或 `localStorage`，如果这两个也是空的，代码里有一个兜底：

```js
APP_DATA.products = mergedProducts.length > 0 ? mergedProducts : getDefaultProducts();
```

`getDefaultProducts()` 是硬编码在 `admin.js` 里的 **5 个占位产品**（PE袋子、防雾膜等示例数据）。**这就是"5"这个数字的来源。**

### 4. 真正致命的一步：**读取失败后，代码会自动把这份"缩水数据"写回磁盘**
在旧版 `selectFolder()` 里：

```js
var diskReadOk = await readProductsDataFromFolder();
if (diskReadOk) {
  ...
} else {
  // 磁盘文件不存在或解析失败，将当前后台数据写入创建它
  await writeProductsDataToDisk(true);   // ← 灾难发生点
}
```

只要"从磁盘读取"这一步失败（哪怕只是选错了文件夹、或者文件被其他工具改过一点格式），代码不会报错阻止，而是**直接把当前只有 5 个产品的 APP_DATA 写盘，永久覆盖掉 aec-project/products-data.js 里原本 256 个产品的真实数据**。写盘没有任何"数量校验"或"备份"机制，覆盖了就是覆盖了，无法撤销。

这也同时解释了产品详情页打不开的问题：`product.html?id=123` 靠 `product-detail-render.js` 读 `window.productsData.products` 按 id 查找，磁盘上的 `products-data.js` 一旦被 5 条数据覆盖，其余 251 个 id 全部查不到，页面自然空白。

**结论：这不是偶然的 bug，是架构上"多份数据源 + 无校验直接覆盖磁盘"的必然结果 —— 只要触发条件出现一次，就会永久丢失数据。之前每次让你重新爬取，其实都是在往这个"漏水的桶"里加水。**

## 二、这次修复做了什么（彻底堵住这个漏洞，而不是再修一次表面症状）

我没有只是"把产品数据加回来"，而是重新设计了保存链路，核心原则：**磁盘写入必须经过唯一的安全网关，绝不允许无校验的静默覆盖。**

新增文件 `products-sync-guard.js`（后台首页和产品编辑页共用同一份，不再各写各的写盘逻辑）：

1. **危险下降拦截**：本机会记住"历史见过的最大产品数量"。任何一次准备写盘的数据，如果比历史记录少了超过 20%（且历史记录 ≥10，避免影响你刚建站时的正常小数据量），会**直接拒绝写盘**，弹窗把原因讲清楚（很可能是文件夹没连对/缓存被清空），需要你手动确认"我已核实，仍要覆盖"才会强制写入。
2. **自动备份**：以后每次真正写盘前，会先把磁盘上*现有*的 `products-data.js` 另存一份到 `aec-project/backups/products-data.<时间戳>.js`，自动保留最近 15 份。就算真出问题，也能从 backups 里一键找回，不再是"死无对证"。
3. **选择文件夹后不再自动写盘**：以前"选文件夹→读取失败→自动初始化写盘"这一步已删除，改成只提示、不动磁盘文件，把决定权交还给你。
4. **加载时的健康提示**：如果这次加载到的产品数量远低于历史记录，后台顶部会出现醒目红色横幅，提醒你"当前数据可能不完整，保存前请先核实"，不会再让你在不知情的情况下点了保存/同步就把真实数据冲掉。
